# Curated Examples

This folder contains curated formatting examples for OPTIMADE property-definition source YAML files. These examples are not exhaustive.

The magnetic moment example is useful because it demonstrates units, inheritance, nested item structure, nullable list values, and provider-style description formatting.

Consult the upstream repositories for broader standard patterns when available:

- `external/OPTIMADE`
- `external/optimade-property-tools`

- `_exmpl_symop.yaml` is useful because it demonstrates the preferred dictionary-property style: `|-` descriptions with one sentence per source line, the `It MUST be a dictionary with the following keys:` requirements pattern, markdown subitems with REQUIRED/OPTIONAL and type phrases, `x-optimade-dimensions`, and exact fraction vectors as lists of strings.
